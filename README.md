# Lost Ark Raid Builder

로스트아크 공격대 자동구성 웹앱입니다. 공격대별로 플레이어, 원정대, 캐릭터, 레이드 선택과 주간 완료 현황을 공유하고 조건을 만족하는 공격대 편성을 자동으로 만듭니다.

## 동작 방식

- 팀장이 공격대 이름과 비밀번호를 정해 공유 공격대를 생성합니다.
- 같은 공격대 이름과 비밀번호로 가입한 사용자는 멤버 목록과 레이드 현황을 함께 편집합니다.
- 공유 데이터는 Cloudflare D1에 저장되고, 다른 사용자의 변경 사항은 약 5초 안에 반영됩니다.
- 레이드 현황에서 완료한 레이드를 체크하면 자동구성 대상에서 제외됩니다.
- 완료 상태는 매주 수요일 오전 6시(KST)에 새 주차로 자동 전환되며 수동 전체 초기화도 가능합니다.
- Lost Ark API 키는 우상단 톱니바퀴 설정에서 각 사용자가 직접 입력합니다.
- API 키는 서버에 저장하지 않고 브라우저 저장소에만 보관합니다.
- 기존 브라우저 저장 자료가 있으면 새 공격대 생성 시 선택해서 가져올 수 있습니다.

## 로컬 실행

필수 환경:

- Node.js `>=22.13.0`

명령:

```bash
npm install
npm run dev
```

프로덕션 빌드 확인:

```bash
npm run build
```

## Cloudflare 배포

이 앱은 `/api/lostark/roster`와 D1을 사용하므로 GitHub Pages 같은 정적 호스팅에는 맞지 않습니다. Cloudflare Workers와 D1을 사용합니다.

- `.openai/hosting.json`의 논리 D1 바인딩 이름은 `DB`입니다.
- `vite.config.ts`에는 운영 D1 데이터베이스 이름과 ID가 연결되어 있습니다.
- 최초 환경에서는 `drizzle/0000_lame_shard.sql`을 원격 D1에 적용한 뒤 배포합니다.
- 이후 `npm run deploy:cloudflare`로 빌드와 Workers 배포를 실행합니다.
- 운영 반영 전에는 `npm run build`와 localhost 동작을 먼저 확인합니다.

## 배포 후 확인

- 공개 URL에서 공격대 생성과 가입 화면이 열리는지 확인합니다.
- 다른 브라우저에서 같은 이름과 비밀번호로 가입해 멤버 목록이 공유되는지 확인합니다.
- 톱니바퀴 설정에서 Lost Ark API 키를 입력합니다.
- 대표 캐릭터 동기화가 동작하는지 확인합니다.
- 레이드 완료 체크가 다른 브라우저에 반영되고 자동구성에서 제외되는지 확인합니다.
- 자동구성 결과가 기존 제약조건대로 생성되는지 확인합니다.
